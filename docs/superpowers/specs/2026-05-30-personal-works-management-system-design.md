## 1. 引言

### 1.1 目的

本文档旨在明确个人著作信息管理系统的功能需求、非功能需求、接口需求、安全性需求及约束条件，作为系统设计、开发、测试和验收的依据。

系统覆盖个人著作管理的完整生命周期，包括著作信息的增删改查、多维度统计图表展示、系统账户管理及数据字典维护。

## 2. 总体描述

### 2.1 产品概述

个人著作信息管理系统是一个基于 Spring Boot + Vue 前后端分离的 Web 应用，支持个人著作信息的管理、统计分析和系统配置维护。

### 2.2 用户角色

| 角色 | 描述 | 主要功能 |
|------|------|---------|
| 普通用户 | 一般用户 | 个人著作管理、个人著作统计 |
| 系统管理员 | 维护系统配置 | 系统管理（账户 + 数据字典） |

### 2.3 系统上下文

系统后端通过 API 网关对外暴露 RESTful 接口，前端通过网关调用后端服务，数据库采用 MySQL 存储数据。

## 3. 功能需求

### 3.1 个人著作管理

#### 3.1.1 著作信息列表

- 以表格形式展示所有著作信息，支持分页
- 列表字段：工号、姓名、著作类型、名称、发表路径、状态、个人排名、著作人、取得时间、备注
- 支持按姓名、著作类型、名称、状态进行多条件筛选查询

#### 3.1.2 新增著作

- 表单字段：
  - 工号（文本输入，必填）
  - 姓名（文本输入，必填）
  - 著作类型（下拉选择，必填，数据来源：数据字典）
  - 名称（文本输入，必填）
  - 发表路径（文本输入，选填）
  - 状态（下拉选择，必填，数据来源：数据字典）
  - 个人排名（数字输入，必填，> 0 整数）
  - 著作人（文本输入，必填）
  - 取得时间（日期选择，必填，YYYY-MM-DD，日历控件，统计时按年份聚合）
  - 备注（文本域，选填）
- 表单校验规则：必填项校验、个人排名 > 0 整数校验、日期格式校验

#### 3.1.3 修改著作

- 基于现有记录加载至编辑表单，修改后保存
- 校验规则同新增

#### 3.1.4 删除著作

- 支持单条删除
- 删除前弹出二次确认提示

#### 3.1.5 著作详情查看

- 点击列表行或详情按钮，展示完整著作信息

### 3.2 个人著作统计

#### 3.2.1 筛选条件

- 姓名：下拉选择 + 手动输入，下拉数据来自已录入的著作记录
- 著作类型：下拉选择，数据来自数据字典
- 两个条件可组合筛选

#### 3.2.2 统计图表

筛选后展示以下四类图表：

| 图表类型 | 展示内容 | 图表形式 |
|---------|---------|---------|
| 按著作类型 | 各著作类型的数量分布 | 柱状图/饼图 |
| 按状态 | 各状态的数量分布 | 柱状图/饼图 |
| 按排名 | 个人排名分布（排名区间） | 柱状图 |
| 按取得年份 | 各年份的著作数量 | 折线图/柱状图 |

### 3.3 系统管理

#### 3.3.1 账户管理

- 账户列表：展示所有系统账户，支持分页
- 新增账户：用户名、密码、角色（普通用户/管理员）
- 编辑账户：修改用户名、角色、重置密码
- 删除账户：二次确认后删除
- 密码加密存储

#### 3.3.2 数据字典管理

- 字典类型管理：支持两类字典——著作类型、状态
- 字典项管理：对每个字典类型维护其选项列表
  - 著作类型初始值：科技论文、管理论文、发明专利、国防专利、实用新型专利
  - 状态初始值：已提交、已审理、已公开、已授权、已驳回、已投稿、已录用、已发表
- 支持字典项的新增、编辑、删除
- 字典变更后实时生效

## 4. 非功能需求

### 4.1 性能需求

| 需求项 | 指标 |
|--------|------|
| 并发用户数 | ≥ 10 人同时在线 |
| 页面响应时间 | ≤ 3 秒（95% 情况下） |
| 接口响应时间 | ≤ 1 秒（95% 情况下） |

### 4.2 用户界面

- 前端皮肤可切换：科技风、简略风两种主题
- 响应式布局适配主流浏览器（Chrome/Firefox/Edge）
- 操作流程简洁，符合用户习惯
- 日期字段使用日历控件选择

### 4.3 可用性

- 系统 7×24 小时可用，计划内维护除外
- 关键操作（增删改）有明确操作成功/失败提示

### 4.4 可维护性

- 后端采用分层架构（Controller → Service → DAO）
- 代码遵循 Java 编码规范
- 数据库变更提供版本化管理（Flyway 或手动 SQL 脚本）

---

## 5. 安全性需求

### 5.1 传输安全

- 前后端通信使用 HTTPS 协议
- 敏感数据（如密码）传输时进行 AES 加密

### 5.2 认证与授权

- 用户登录需用户名 + 密码认证
- 密码采用 BCrypt 加密存储
- 基于 JWT Token 进行会话管理
- 接口权限控制：系统管理功能仅管理员可访问

### 5.3 API 网关

- 所有前后端接口通过 API 网关统一管理
- 网关负责：请求路由、认证鉴权、限流、日志记录

### 5.4 数据安全

- 数据库访问使用参数化查询，防止 SQL 注入
- 接口参数校验，防止 XSS 攻击
- 用户操作日志记录关键操作（增删改）

## 6. 接口需求

### 6.1 前端-后端接口

- 全部采用 RESTful 风格 API
- 请求/响应格式：JSON
- 统一响应体结构：`{ code: 200, data: {}, message: "success" }`
- 接口路径前缀：`/api/v1/`

### 6.2 核心 API 列表

| 模块 | 接口 | 方法 | 说明 |
|------|------|------|------|
| 著作管理 | `/api/v1/works` | GET | 分页查询著作列表 |
| 著作管理 | `/api/v1/works` | POST | 新增著作 |
| 著作管理 | `/api/v1/works/{id}` | PUT | 修改著作 |
| 著作管理 | `/api/v1/works/{id}` | DELETE | 删除著作 |
| 著作管理 | `/api/v1/works/{id}` | GET | 查询著作详情 |
| 统计 | `/api/v1/statistics` | GET | 获取统计数据 |
| 字典 | `/api/v1/dict/types` | GET | 获取字典类型列表 |
| 字典 | `/api/v1/dict/items/{type}` | GET | 获取字典项列表 |
| 字典 | `/api/v1/dict/items` | POST | 新增字典项 |
| 字典 | `/api/v1/dict/items/{id}` | PUT | 修改字典项 |
| 字典 | `/api/v1/dict/items/{id}` | DELETE | 删除字典项 |
| 账户 | `/api/v1/users` | GET | 分页查询账户列表 |
| 账户 | `/api/v1/users` | POST | 新增账户 |
| 账户 | `/api/v1/users/{id}` | PUT | 修改账户 |
| 账户 | `/api/v1/users/{id}` | DELETE | 删除账户 |
| 账户 | `/api/v1/auth/login` | POST | 用户登录 |
| 账户 | `/api/v1/auth/logout` | POST | 用户登出 |
